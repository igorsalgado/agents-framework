# Skill: Tailwind UI

## Quando usar
Use esta skill para acelerar a construção visual com consistência, responsividade e baixo custo de manutenção.

## Padrão Sênior
- Use Tailwind para materializar hierarquia visual e espaçamento com rapidez.
- Padronize classes recorrentes em componentes ou utilitários quando a repetição ficar alta.
- Use bibliotecas complementares apenas quando reduzirem esforço de forma clara.
- Priorize clareza visual, contraste e legibilidade antes de efeito visual.

## Boas Práticas
- Definir convenções de spacing, tipografia e cores.
- Reutilizar padrões de layout.
- Combinar Tailwind com componentes de propósito claro.
- Tratar responsividade como requisito base.

## O que evitar
- Colagens gigantes de classes sem abstração mínima.
- Design inconsistente por falta de regra.
- Uso de componente de biblioteca sem entender comportamento e acessibilidade.
- Customização excessiva que anula o ganho da stack.
