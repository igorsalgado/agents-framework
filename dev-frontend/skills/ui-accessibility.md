# Skill: UI Accessibility

## Quando usar
Use esta skill para garantir que componentes, fluxos, formulários e navegação sejam acessíveis e operáveis em cenários reais.

## Padrão Sênior
- Priorizar HTML semântico e comportamento nativo.
- Garantir ordem de foco, leitura por teclado e feedback perceptível.
- Tratar mensagens, validação e estados dinâmicos de forma acessível.
- Validar padrões ARIA somente quando semântica nativa não resolver.

## Checklist Base
- Navegação por teclado funcional.
- Nome acessível em controles e regiões importantes.
- Contraste e foco visíveis.
- Erros associados ao campo correto.
- Compatibilidade com zoom, reflow e leitores de tela.

## O que evitar
- Div ou span simulando botão.
- Uso de ARIA para esconder semântica ruim.
- Modal sem foco controlado.
- Toast, spinner ou aviso sem comunicação adequada.
