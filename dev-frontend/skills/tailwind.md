# Skill: Tailwind

## Quando usar
Use esta skill para acelerar construção visual com consistência, utilitários previsíveis e baixo custo de manutenção em times que precisam entregar interface sem inflar CSS customizado.

## Padrão Sênior
- Usar Tailwind para compor layout, espaçamento, tipografia e estados com consistência.
- Extrair padrões recorrentes para componentes ou abstrações leves quando a repetição começar a esconder intenção.
- Manter tokens, responsividade e estados visuais explícitos.
- Usar bibliotecas complementares apenas quando reduzirem esforço recorrente de verdade.
- Preferir clareza de composição a listas de classes sem semântica operacional.

## Checklist Base
- Hierarquia visual consistente.
- Estados visuais de hover, foco, erro e desabilitado definidos.
- Responsividade tratada nos pontos críticos.
- Repetição excessiva identificada e reduzida com critério.
- Acessibilidade preservada em componentes customizados.

## O que evitar
- Empilhar classes sem entender o resultado final.
- Criar abstrações cedo demais para esconder utilitários simples.
- Perder consistência de tokens, cores e espaçamento.
- Usar Tailwind como substituto de arquitetura de componentes.
- Componentes visualmente corretos, mas sem foco, contraste ou semântica adequados.
