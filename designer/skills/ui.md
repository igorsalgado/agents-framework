# Skill: UI

## Quando usar
Use esta skill para definir aparência, hierarquia visual, composição, densidade de informação e comportamento visual da interface.

## Padrão Sênior
- Construir hierarquia visual em função da tarefa principal.
- Usar espaçamento, contraste, tipografia e ritmo para orientar leitura.
- Tratar componentes como contratos com estados bem definidos.
- Especificar responsividade e comportamento, não apenas aparência estática.

## Critérios
- Um usuário novo entende prioridade e ação principal em poucos segundos.
- A composição funciona em desktop e mobile sem colapso de legibilidade.
- Estados interativos são distinguíveis e consistentes.
- Tokens visuais são reutilizáveis e não arbitrários.

## O que evitar
- Layout “bonito” mas sem prioridade visual.
- Tipografia sem escala clara.
- Cores sem função semântica.
- Componentes com estados incompletos.
