# Skill: Design Systems

## Quando usar
Use esta skill para padronizar componentes, tokens, nomenclatura e governança entre design e engenharia.

## Padrão Sênior
- Pensar em sistema antes de tela isolada.
- Definir tokens antes de valores pontuais.
- Tratar cada componente como API com variantes, estados, restrições e exemplos.
- Documentar quando reutilizar, quando estender e quando criar algo novo.

## Estrutura Mínima
- Tokens: cor, tipografia, espaçamento, raio, sombra e motion.
- Componentes: propósito, anatomia, variantes, estados e acessibilidade.
- Regras de contribuição: critérios para entrar, alterar ou depreciar componentes.
- Handoff: mapeamento claro entre design e implementação.

## O que evitar
- Biblioteca inchada por exceções locais.
- Componentes sem limite de uso.
- Divergência entre design e código sem rastreabilidade.
- Variantes criadas para “resolver rápido” sem governança.
